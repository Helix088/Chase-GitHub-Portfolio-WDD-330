Notes

  1. Client-side validation is an initial check and an important feature of good user experience; by catching invalid data on the client-side, the user can fix it straight away.
  2. Client-side validation is put into either the HTML itself or in the JavaScript. With HTML the validation is built in with the form tag and with JavaScript you use the Constraint Validation API.
  3. In HTML it is built into the form element where you can use attributes to tell the form to require a input in a input, make the input a certain length, or have the input have a certain pattern. In JavaScript you can use the Constraint Validation API, which will give you access to different methods that can do the same thing in HTML, but are a lot more customizable.
  4. The Fetch API provides a JavaScript interface for accessing and manipulating parts of the protocol, such as requests and responses. It also provides a global fetch() method that provides an easy, logical way to fetch resources asynchronously across the network.
  5. How do you use Fetch API? To use the Fetch API, you use the fetch method along with the JSON source path, but this does not grab the JSON content. In order to get the content of the JSON you use the .json() method.
  6. Why do we use Fecth API? Fetch provides a better way that can be used by other technologies. It also provides a single logical place to define other HTTP-related concepts.
